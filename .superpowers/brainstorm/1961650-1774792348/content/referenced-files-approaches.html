How should referenced files be accessible?

The global CLAUDE.md references files like UI-STANDARDS.md, WINDOWS-BUILD-VM.md, SOLUTIONS.md, etc. Currently there's no way to edit them from the Workbench UI. Here are 3 approaches — click the one that feels right.

A

Inline Links in the Editor

When you open the global CLAUDE.md, any ~/.claude/SOMETHING.md path in the text becomes a clickable link that opens that file in a new floating editor window. Zero UI chrome added — the discoverability is in the content itself.

Pros

  • No new UI elements needed
  • Works automatically for ANY referenced file
  • Natural — you see the reference, you click it
  • Scales to any number of files

Cons

  • Only discoverable when CLAUDE.md is open
  • Requires parsing markdown content for file paths
  • CodeMirror link detection adds complexity
B

Expand the File List (Backend Discovery)

The backend already lists CLAUDE.md files and rules/*.md. Extend it to also list all .md files under ~/.claude/ (like UI-STANDARDS.md, WINDOWS-BUILD-VM.md, etc.) as a new "global-files" category. They show up in the existing sidebar, command palette, and anywhere CLAUDE.md files are listed.

Pros

  • Files always visible in sidebar without opening CLAUDE.md first
  • Works with existing UI patterns (sidebar, command palette)
  • Simple backend change — just glob more files
  • No frontend parsing logic needed

Cons

  • Shows ALL .md files, even ones you rarely edit
  • Sidebar could get cluttered with many global files
  • Doesn't distinguish "referenced by CLAUDE.md" from "happens to exist"
C

Both: Backend Discovery + Clickable Links

List all ~/.claude/*.md files in the sidebar/command palette (Option B), AND make file paths clickable inside the CodeMirror editor (Option A). Two entry points for the same action.

Pros

  • Maximum discoverability — find files from sidebar OR from within content
  • Each piece is independently useful
  • Clickable paths useful in project CLAUDE.md files too

Cons

  • More implementation work (both backend + CodeMirror plugin)
  • May be overkill for how often these files are edited
